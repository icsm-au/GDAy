To transform multiple folders of files

  1. On the File menu, select Transform Folder.
  2. Click Look in.
  3. From the explorer tree, either click the drive or root folder that contains the input files or type in the full folder path in the Folder box, and then click OK.
  4. From the Files format box in the Input Folder group, choose the extension that matches the input file-type you are working with (note: the available types include Formatted Text (*.dat, *.prn, *.txt), Comma Separated Values (*.csv) files), SCDB files (*.man), NEWGAN res files (*.res), and Queensland Interchange Format (*.qif)).
  5. If the selected coordinate system parameters specify that the input coordinates are geographic coordinates, then select the appropriate format type from the Geographic Format box.
  6. Click Search for files in all subfolders if you wish to transform all other subfolders contained within the specified folder.
  7. Click Save in.
  8. From the explorer tree, either click the drive or root folder that contains the output files or type in the full folder path in the Folder box, and then click OK .
  9. From the Files format box in the Output Folder group, choose the estension that matches the output file-type you are working with (note: the available types displayed will reflect the allowable file translations depending on the selected input file format).
  10. If the selected coordinate system parameters specify that the input coordinates are geographic coordinates, then select the appropriate format type from the Geographic Format box.
  11. If the output file format will be either a formatted text (*.txt) or comma separated values (*.csv) file, click Write header to dat/csv output files if you wish to have a standard header printed at the start of every output file.
  12. Click OK.

 

Important

  • Before attempting to transform a folder of files, ensure that you have selected the right coordinate system parameters and that all files in the input folder correspond with the selected coordinate type, ie. decimal degrees; degrees minutes and seconds; or HP notation.

Notes

  • Since GDAy assumes that all files in a folder will be of the same coordinate type, this method of transforming files requires you to order your files into subfolders of the same coordinate type, such as decimal degrees or UTM coordinates.
  • You can not use this multiple file transformation function to transform files that contain coordinates of different types.
  • If the specified output folder is the input folder, the output file names will be exactly that of the input file names with a ‘_’ character inserted at the front of the file name. For example, if the input folder contains a file called “GDA94 HP1.dat”, the output file name will be “_GDA94 HP1.dat”.
  • If you are transforming a Queensland Interchange Format file (*.qif) and the coordinates to be printed to the output file are projection coordinates, you will be prompted to specify a zone for the output file. Specifying a unique zone will ensure that all transformed coordinates will be projected over a single zone, regardless of how far east and west the file spans.
  • After the files have been successfully transformed, you will be prompted to transform another folder. If at least one point in an input file could not be transformed, or an error occurred in the file-read/write process, the particular error details and the input coordinates are printed to the Log File.
  • If any of the points within a file lie outside the limits of the distortion grid, they will be transformed using a 7-Parameter conformal transformation. In this case, the transformation parameters selected from the Select Distortion Grid dialog box will be used.
  • You can view the Log file by selecting View Log File on the Tools menu.
  • If you want to transform latitudes and longitudes and the Separated DMS fields and HP Notation option is transparent, then you will have to go back to the Coordinate System dialog box to select the right coordinate type.
