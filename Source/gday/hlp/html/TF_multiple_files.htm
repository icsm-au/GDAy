To transform multiple files

  1. On the File menu, select Transform Points File.
  2. Click Browse.
  3. In the Look in box, click the drive or folder that contains the input file.
  4. In the Files of type box, choose the appropriate input file-type (note: the available types include Formatted Text (*.dat, *.prn, *.txt), Comma Separated Files (*.csv) files), SCDB files (*.man), NEWGAN res files (*.res), and Queensland Interchange Format (*.qif).
  5. In the folder list, double-click folders until you open the folder that contains the required files.
  6. Highlight all the input files you want to use, and then click OK.
  7. If you are transforming a file of geographic coordinates in degrees, minutes and seconds, select the appropriate radio buttons from the Input and Output File Groups. This radio button option is greyed when you are transforming SCDB files (*.man), NEWGAN Res files (*.res) or Queensland Interchange Format files (*.qif).
  8. Click OK.

 

Important

  • Before attempting to transform multiple files, ensure that you have selected the right coordinate system parameters .

Notes

  • You can not use this multiple file transformation function to transform files that contain coordinates of different types. Similarly, all input files must be of the same file format. If you are transforming multiple Formatted Text or Comma Separated files that contain geographic coordinates, ensure that they are all in the same format, ie. decimal degrees; degrees minutes and seconds; or HP notation.
  • Upon selecting multiple files in step 6, the output File Name edit box and Save As button are greyed to restrict you from output file selection. For every file you selected, the output file names will be exactly that of the input file names with a ‘_’ character inserted at the front of the file name. For example, if you selected the files “GDA94 HP1.dat” and “GDA94 HP2.dat” the output file names will be “_GDA94 HP1.dat” and “_GDA94 HP2.dat” respectively.
  • If you are transforming a Queensland Interchange Format file (*.qif) and the coordinates to be printed to the output file are projection coordinates, you will be prompted to specify a zone for the output file. Specifying a unique zone will ensure that all transformed coordinates will be projected over a single zone, regardless of how far east and west the file spans.
  • After the files have been successfully transformed, you will be prompted to transform another file. If at least one point in the input file could not be transformed, or an error occurred in the file-read/write process, the particular error details and the input coordinates are printed to the Log File.
  • If any of the points within the file lie outside the limits of the distortion grid, they will be transformed using a 7-Parameter conformal transformation. In this case, the transformation parameters selected from the Select Distortion Grid dialog box will be used. For more information, see “Transformation Parameters” from Choosing a distortion grid file.
  • For formatted text and comma separated files only, the corresponding interpolation accuracy values are printed at the end of each line in the output file. If the accuracy values could not be determined, 3 dashes: “ --- “ will be printed instead.
  • You can view the Log file by selecting View Log File on the Tools menu.
  • If you want to transform latitudes and longitudes and the Separated DMS fields and HP Notation option is transparent, then you will have to go back to the Coordinate System dialog box to select the right coordinate type.
