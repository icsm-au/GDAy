Supported file types

GDAy supports Formatted text files (*.dat, *.prn, *.txt), Comma separated files (*.csv), SCDB files (*.man), NEWGAN res files (*.res) and Queensland Interchange Format (QIF) files (*.qif). For the technical specifications on the format of SCDB, NEWGAN res and QIF files, contact the Geodetic Survey Section, Department of Natural Resources, QLD.

It is not recommended that you attempt to create SCDB or QIF files for your input coordinates. The facility to transform SCDB and QIF files is for the purpose of transforming those files issued from the Department of Natural Resources, QLD only.

When transforming formatted text and comma separated files, you can specify your input coordinate type to be any one of the available types. For files that contain geographic coordinates, you can format the latitude and longitude values using one of five types: Decimal Degrees, DMS HP Notation, Separated DMS Fields, DMIN HP Notation, and Separated DMIN Fields. See Formatting options for geographic coordinates for more information on the accepted formatting types.

An important note to make is that you can not have different coordinate types in the input file to be transformed. For example, when you are transforming geographic coordinates, you can not have latitude and longitude on one record and easting, northing and zone on another. Similarly, if you are transforming a file of geographic coordinates, you must use only one type of format for all values within the file.

 

Notes

  • You do not need to have a header record within the file. For every record that is read from the file that does not contain valid coordinates, the record is printed to the log file. This includes any textual nomenclature or narrations within the file.
  • For SCDB (*.man) files, you do not need to worry about selecting the coordinates type or whether it is in decimal degrees, HP Notation or Separated DMS fields for either input or output files.
  • For NEWGAN res (*.res) and QIF (*.qif) files, you must select the correct coordinates type for the input and output files. If you have selected the wrong input coordinates type, GDAy will halt the file transformation and prompt you with an error message. If either *.res or *.qif files are in geographic coordinates, you don’t have to worry about whether it is in decimal degrees, HP Notation or Separated DMS fields for either input or output files. Since all geographic data within *.qif files are in decimal degrees, and all geographic data within *.res files are in degrees, minutes and seconds, this option is overridden.
