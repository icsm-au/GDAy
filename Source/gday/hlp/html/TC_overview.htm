Transforming coordinates overview

When using GDAy, there are three main steps to transforming coordinates. Click on the steps below:

1.  Select the distortion grid file

When you want to transform coordinates from one datum to another, the recommended approach is to perform a high accuracy grid transformation, or distortion modelling: the process of interpolating known correction values from a grid file. GDAy incorporates this method by using an NTv2 distortion grid file. For more information, see High accuracy grid transformation.

To ensure that you will be achieving the most accurate coordinate transformations as possible, you need to select a distortion grid that covers an area your points are within.

To select a distortion grid, see Choosing a distortion grid file.

Notes

  • The Setup program will allow you to select the latest version of the Queensland Distortion Grid (QLD_0900.gsb) as the default grid. If you choose not to use the Queensland Distortion Grid during Setup, when you run GDAy for the first time you will need to select the desired grid file that covers your area of interest.
  • Once an appropriate distortion grid file has been selected, this file will remain as the default distortion grid until you select a new file location.

2.  Select the input and output coordinate system parameters

In broad terms, datum transformations facilitate the conversion of coordinates from one datum to another. The process is dependent on a mathematical relationship, which can be thought of as the spatial difference between two different datum of interest. In practice, the relationship is computed from the difference between two coordinate sets based on the respective datum. GDAy supports the transformation of coordinates based on the GDA.

Similarly, if you require the conversion of coordinates from geographic to projection, you will also need to select the appropriate input and output coordinate types. Converting coordinates from one type to another on the same datum does not require a distortion grid.

To select the appropriate input and output coordinate system settings, see Choosing the right coordinate system parameters.

Notes

  • Due to the spatial difference between GDA94 and GDA2020, it is imperitive that you select the appropriate input and output datum.
  • Once you have selected the input and output coordinate system parameters, GDAy will retain these settings until you decide to select new settings.

3.  Performing a coordinate transformation

GDAy enables you to transform coordinates using two ways: Interactive transformations and File transformations.

Interactive transformations
Transforming coordinates using this method is performed on the main GDAy dialog and can be used to transform a single point or a small list of points. Simply enter the coordinate values for your points into the input edit boxes on the left hand side of the main GDAy dialog and press Compute. The transformed coordinates will then be displayed in the output edit boxes on the right.

To transform coordinates using this method, see Transforming a single point.

File transformations
Transforming coordinates using this method requires the list of points you wish to transform to be placed in a file using a structured format. If you are working with large numbers of files of the same file format, you may choose to transform multiple files or whole folders of files in one sinle step. There are several options for transforming files of coordinates, each of which will require you to be aware of the coordinate system parameters and the type of file you are working with.

To transform a file of coordinates, see Transforming Files.

Important

  • Before attempting to transform coordinates using either of these methods, ensure that you have selected the right coordinate system parameters outlined in Step 2. above.
